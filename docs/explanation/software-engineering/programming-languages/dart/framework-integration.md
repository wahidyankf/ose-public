# Dart Framework Integration Standards

## Prerequisite Knowledge

**REQUIRED**: You MUST understand Dart fundamentals from [AyoKoding Dart Learning Path](../../../../../apps/ayokoding-web/content/en/learn/software-engineering/programming-languages/dart/_index.md) before using these standards.

**This document is OSE Platform-specific**, not a Dart tutorial. We define HOW to integrate frameworks in THIS codebase, not WHAT Flutter or Riverpod are.

**See**: [Programming Language Documentation Separation Convention](../../../../../governance/conventions/structure/programming-language-docs-separation.md)

## Purpose

This document defines **authoritative framework integration standards** for Dart development in the OSE Platform. Consistent framework usage patterns reduce bugs, improve testability, and ensure maintainable Flutter and server-side Dart applications.

**Target Audience**: OSE Platform Dart developers building Flutter apps or shelf HTTP services

**Scope**: Flutter widget patterns, Riverpod state management, shelf server setup with Router and Pipeline, shelf_router for REST APIs, dio interceptors

## Software Engineering Principles

### 1. Automation Over Manual

**PASS Example** (Riverpod Code Generation):

```dart
// Use riverpod_generator for automated provider generation
// Run: dart run build_runner watch

import 'package:riverpod_annotation/riverpod_annotation.dart';

part 'zakat_providers.g.dart'; // Generated by build_runner

// @riverpod generates type-safe providers automatically
@riverpod
class ZakatCalculationNotifier extends _$ZakatCalculationNotifier {
  @override
  ZakatCalculationState build() => const ZakatCalculationState.initial();

  Future<void> calculate(double wealth, double nisab) async {
    state = const ZakatCalculationState.loading();
    try {
      final amount = await ref.read(zakatServiceProvider).calculate(wealth, nisab);
      state = ZakatCalculationState.success(amount: amount);
    } on ZakatValidationException catch (e) {
      state = ZakatCalculationState.error(message: e.message);
    }
  }
}
```

**PASS Example** (Explicit Widget Keys):

```dart

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        TextField(key: wealthInputKey, /* ... */),
        TextField(key: nisabInputKey, /* ... */),
        ElevatedButton(
          key: calculateButtonKey,
          onPressed: () {},
          child: const Text('Calculate'),
        ),
      ],
    );
  }
}
```

### 3. Immutability Over Mutability

**PASS Example** (Immutable Widget State):

```dart
// CORRECT: StatelessWidget with immutable state via Riverpod
class ZakatSummaryCard extends ConsumerWidget {
  // All fields final
  final String customerId;

  const ZakatSummaryCard({required this.customerId, super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    // State from provider - not mutable local state
    final summary = ref.watch(zakatSummaryProvider(customerId));

    return summary.when(
      data: (s) => _buildSummaryCard(s),
      loading: () => const CircularProgressIndicator(),
      error: (e, _) => Text('Error: $e'),
    );
  }
}
```

### 4. Pure Functions Over Side Effects

**PASS Example** (Pure Widget Build Methods):

```dart
// CORRECT: Pure build method - same state always produces same UI
class ZakatAmountDisplay extends StatelessWidget {
  final double amount;
  final String currency;

  const ZakatAmountDisplay({
    required this.amount,
    required this.currency,
    super.key,
  });

  @override
  Widget build(BuildContext context) {
    // Pure: same amount + currency -> same widget tree
    return Text(
      '${amount.toStringAsFixed(2)} $currency',
      style: Theme.of(context).textTheme.headlineMedium,
    );
  }
}
```

### 5. Reproducibility First

**PASS Example** (Testable Provider Overrides):

```dart
// CORRECT: Override providers in tests for reproducible results
testWidgets('shows zakat amount after calculation', (tester) async {
  await tester.pumpWidget(
    ProviderScope(
      overrides: [
        // Override with deterministic mock
        zakatServiceProvider.overrideWith(
          (ref) => MockZakatService(fixedAmount: 250.0),
        ),
      ],
      child: const MaterialApp(home: ZakatCalculatorScreen()),
    ),
  );

  // Deterministic test - same mock produces same UI
  await tester.tap(find.byKey(ZakatCalculatorForm.calculateButtonKey));
  await tester.pumpAndSettle();
  expect(find.text('250.00 USD'), findsOneWidget);
});
```

## Part 1: Flutter Widget Patterns

### StatelessWidget Over StatefulWidget

**MUST** prefer `StatelessWidget` and Riverpod providers over `StatefulWidget` for state management.

```dart
// CORRECT: StatelessWidget with Riverpod for state
class ZakatHistoryScreen extends ConsumerWidget {
  const ZakatHistoryScreen({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final history = ref.watch(zakatHistoryProvider);

    return Scaffold(
      appBar: AppBar(title: const Text('Zakat History')),
      body: history.when(
        data: (transactions) => ZakatHistoryList(transactions: transactions),
        loading: () => const Center(child: CircularProgressIndicator()),
        error: (error, stack) => ErrorView(error: error),
      ),
    );
  }
}

// WRONG: StatefulWidget for data that belongs in a provider
class ZakatHistoryScreenBad extends StatefulWidget {
  const ZakatHistoryScreenBad({super.key});

  @override
  State<ZakatHistoryScreenBad> createState() => _ZakatHistoryScreenBadState();
}

class _ZakatHistoryScreenBadState extends State<ZakatHistoryScreenBad> {
  List<ZakatTransaction>? _transactions; // State that should be in a provider!

  @override
  void initState() {
    super.initState();
    _loadTransactions(); // Side effect in widget!
  }

  Future<void> _loadTransactions() async {
    final transactions = await zakatService.loadHistory();
    setState(() => _transactions = transactions); // Manual state management!
  }
}
```

### StatefulWidget When Appropriate

**MUST** use `StatefulWidget` only for UI-local state (animations, form input, focus).

```dart
// CORRECT: StatefulWidget for UI-local animation state
class ZakatResultAnimation extends StatefulWidget {
  final double amount;

  const ZakatResultAnimation({required this.amount, super.key});

  @override
  State<ZakatResultAnimation> createState() => _ZakatResultAnimationState();
}

class _ZakatResultAnimationState extends State<ZakatResultAnimation>
    with SingleTickerProviderStateMixin {
  late final AnimationController _controller;
  late final Animation<double> _fadeAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 500),
    )..forward();
    _fadeAnimation = CurvedAnimation(parent: _controller, curve: Curves.easeIn);
  }

  @override
  void dispose() {
    _controller.dispose(); // Always dispose controllers
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return FadeTransition(
      opacity: _fadeAnimation,
      child: Text('${widget.amount.toStringAsFixed(2)} USD'),
    );
  }
}
```

### Const Constructors in Widget Trees

**MUST** use `const` constructors in widget trees to minimize rebuilds.

```dart
// CORRECT: const wherever possible
@override
Widget build(BuildContext context) {
  return const Scaffold(
    appBar: ZakatAppBar(),           // const
    body: ZakatCalculatorForm(),     // const
    floatingActionButton: ZakatHistoryButton(), // const
  );
}

// WRONG: Missing const (unnecessary rebuilds)
@override
Widget build(BuildContext context) {
  return Scaffold(                   // Missing const
    appBar: ZakatAppBar(),           // Missing const
    body: ZakatCalculatorForm(),     // Missing const
  );
}
```

### Widget Decomposition

**MUST** decompose large widgets into smaller, focused widgets.

```dart
// WRONG: Monolithic widget
class ZakatDashboard extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Column(
        children: [
          // 50+ lines of widget code mixed together
          Container(
            padding: const EdgeInsets.all(16),
            child: Column(
              children: [
                // Zakat summary section
                // Transaction list section
                // Statistics section
                // All mixed in one widget!
              ],
            ),
          ),
        ],
      ),
    );
  }
}

// CORRECT: Decomposed into focused widgets
class ZakatDashboard extends StatelessWidget {
  const ZakatDashboard({super.key});

  @override
  Widget build(BuildContext context) {
    return const Scaffold(
      body: CustomScrollView(
        slivers: [
          ZakatSummarySliver(),     // Focused widget
          ZakatTransactionSliver(), // Focused widget
          ZakatStatisticsSliver(),  // Focused widget
        ],
      ),
    );
  }
}
```

## Part 2: Riverpod State Management

### Provider Types

**MUST** choose the correct provider type:

```dart
// Provider: sync, read-only value
@riverpod
ZakatConfig zakatConfig(ZakatConfigRef ref) {
  return ZakatConfig.fromEnvironment();
}

// FutureProvider: async, single value
@riverpod
Future<List<ZakatTransaction>> zakatHistory(
  ZakatHistoryRef ref,
  String customerId,
) async {
  final service = ref.read(zakatServiceProvider);
  return service.loadHistory(customerId);
}

// StreamProvider: async, multiple values over time
@riverpod
Stream<List<ZakatTransaction>> zakatHistoryStream(
  ZakatHistoryStreamRef ref,
  String customerId,
) {
  final repository = ref.read(zakatRepositoryProvider);
  return repository.watchByCustomerId(customerId);
}

// NotifierProvider: mutable state with methods
@riverpod
class ZakatCalculationNotifier extends _$ZakatCalculationNotifier {
  @override
  ZakatCalculationState build() => const ZakatCalculationState.initial();

  Future<void> calculate({required double wealth, required double nisab}) async {
    state = const ZakatCalculationState.loading();
    try {
      final service = ref.read(zakatServiceProvider);
      final amount = await service.calculate(wealth, nisab);
      state = ZakatCalculationState.success(amount: amount);
    } on ZakatValidationException catch (e) {
      state = ZakatCalculationState.error(message: e.message);
    }
  }

  void reset() {
    state = const ZakatCalculationState.initial();
  }
}
```

### Reading vs Watching Providers

**MUST** use `ref.watch` for reactive UI, `ref.read` for one-time access in callbacks.

```dart
class ZakatCalculatorWidget extends ConsumerWidget {
  const ZakatCalculatorWidget({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    // CORRECT: watch rebuilds widget when state changes
    final calculationState = ref.watch(zakatCalculationNotifierProvider);

    return Column(
      children: [
        _buildStateDisplay(calculationState),
        ElevatedButton(
          onPressed: () {
            // CORRECT: read for one-time action (not in build)
            ref.read(zakatCalculationNotifierProvider.notifier).calculate(
              wealth: _wealthController.text.toDouble(),
              nisab: _nisabController.text.toDouble(),
            );
          },
          child: const Text('Calculate'),
        ),
      ],
    );
  }
}
```

## Part 3: shelf Server Setup

### Complete Server Setup

**MUST** follow this structure for shelf HTTP servers:

```dart
import 'dart:io';
import 'package:shelf/shelf.dart';
import 'package:shelf/shelf_io.dart' as shelf_io;
import 'package:shelf_router/shelf_router.dart';

// Dependency injection container
class AppContainer {
  late final ZakatRepository zakatRepository;
  late final ZakatService zakatService;
  late final MurabahaRepository murabahaRepository;
  late final MurabahaService murabahaService;

  AppContainer() {
    // Wire dependencies
    zakatRepository = SqliteZakatRepository(Database.open('zakat.db'));
    zakatService = ZakatService(repository: zakatRepository);
    murabahaRepository = SqliteMurabahaRepository(Database.open('murabaha.db'));
    murabahaService = MurabahaService(repository: murabahaRepository);
  }
}

// Build the complete request handler
Handler buildApp(AppContainer container) {
  final router = Router();

  // Mount route handlers
  final zakatHandler = ZakatRouteHandler(service: container.zakatService);
  final murabahaHandler = MurabahaRouteHandler(service: container.murabahaService);

  // Register routes
  router.mount('/api/v1/zakat/', zakatHandler.router.call);
  router.mount('/api/v1/murabaha/', murabahaHandler.router.call);
  router.get('/health', (Request _) => Response.ok('OK'));

  // Build middleware pipeline
  return Pipeline()
      .addMiddleware(logRequests())
      .addMiddleware(_corsMiddleware())
      .addMiddleware(_authMiddleware())
      .addMiddleware(_errorHandlerMiddleware())
      .addHandler(router.call);
}

// Start the server
Future<void> main() async {
  final container = AppContainer();
  final handler = buildApp(container);

  final server = await shelf_io.serve(
    handler,
    InternetAddress.anyIPv4,
    int.parse(Platform.environment['PORT'] ?? '8080'),
  );

  print('Zakat API server listening on port ${server.port}');
}
```

### Route Handler Class Pattern

**MUST** group related routes into handler classes.

```dart
class ZakatRouteHandler {
  final ZakatService _service;

  ZakatRouteHandler({required ZakatService service}) : _service = service;

  Router get router {
    final router = Router();

    router.get('/transactions', _listTransactions);
    router.post('/transactions', _createTransaction);
    router.get('/transactions/<id>', _getTransaction);
    router.delete('/transactions/<id>', _deleteTransaction);

    return router;
  }

  Future<Response> _listTransactions(Request request) async {
    final params = request.url.queryParameters;
    final customerId = params['customer_id'];

    if (customerId == null) {
      return _badRequest('customer_id query parameter required');
    }

    final transactions = await _service.listTransactions(customerId);
    return _jsonOk(transactions.map((t) => t.toJson()).toList());
  }

  Future<Response> _createTransaction(Request request) async {
    final body = await _parseJsonBody(request);
    if (body == null) return _badRequest('Invalid JSON body');

    try {
      final input = CreateZakatTransactionRequest.fromJson(body);
      final tx = await _service.createTransaction(input);
      return _jsonCreated(tx.toJson());
    } on ZakatValidationException catch (e) {
      return _badRequest(e.message);
    }
  }

  Future<Response> _getTransaction(Request request, String id) async {
    final tx = await _service.findById(id);
    if (tx == null) return _notFound('Transaction $id not found');
    return _jsonOk(tx.toJson());
  }

  Future<Response> _deleteTransaction(Request request, String id) async {
    await _service.delete(id);
    return Response(204);
  }

  // Response helpers
  Response _jsonOk(Object body) => Response.ok(
        jsonEncode(body),
        headers: {'Content-Type': 'application/json'},
      );

  Response _jsonCreated(Object body) => Response(
        201,
        body: jsonEncode(body),
        headers: {'Content-Type': 'application/json'},
      );

  Response _badRequest(String message) => Response(
        400,
        body: jsonEncode({'error': message}),
        headers: {'Content-Type': 'application/json'},
      );

  Response _notFound(String message) => Response(
        404,
        body: jsonEncode({'error': message}),
        headers: {'Content-Type': 'application/json'},
      );

  Future<Map<String, dynamic>?> _parseJsonBody(Request request) async {
    try {
      return jsonDecode(await request.readAsString()) as Map<String, dynamic>;
    } on FormatException {
      return null;
    }
  }
}
```

## Part 4: dio Interceptors

**SHOULD** implement logging and retry interceptors with dio.

```dart
// Auth + retry interceptor
class ZakatApiInterceptor extends Interceptor {
  final SecureTokenStorage _tokenStorage;
  final int maxRetries;

  ZakatApiInterceptor({
    required SecureTokenStorage tokenStorage,
    this.maxRetries = 3,
  }) : _tokenStorage = tokenStorage;

  @override
  Future<void> onRequest(
    RequestOptions options,
    RequestInterceptorHandler handler,
  ) async {
    final token = await _tokenStorage.getAuthToken();
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    options.headers['X-Request-Id'] = _generateRequestId();
    handler.next(options);
  }

  @override
  Future<void> onError(
    DioException err,
    ErrorInterceptorHandler handler,
  ) async {
    final retryCount = err.requestOptions.extra['retry_count'] as int? ?? 0;

    // Retry on network errors (not 4xx client errors)
    if (_isRetryable(err) && retryCount < maxRetries) {
      final options = err.requestOptions;
      options.extra['retry_count'] = retryCount + 1;

      await Future.delayed(Duration(milliseconds: 200 * (retryCount + 1)));

      try {
        final response = await Dio().fetch(options);
        handler.resolve(response);
        return;
      } on DioException catch (retryError) {
        handler.next(retryError);
        return;
      }
    }

    // Map DioException to domain exceptions
    if (err.response?.statusCode == 401) {
      await _tokenStorage.clearAll();
      handler.next(DioException(
        requestOptions: err.requestOptions,
        error: UnauthorizedException('Session expired'),
      ));
    } else {
      handler.next(err);
    }
  }

  bool _isRetryable(DioException err) {
    return err.type == DioExceptionType.connectionTimeout ||
        err.type == DioExceptionType.receiveTimeout ||
        err.response?.statusCode == 503;
  }

  String _generateRequestId() =>
      DateTime.now().millisecondsSinceEpoch.toRadixString(36);
}
```

## Enforcement

Framework integration standards are enforced through:

- **Widget tests** - Test all UI states (loading, success, error)
- **Provider tests** - Test all state transitions
- **Code reviews** - Verify StatelessWidget preference, const usage, proper provider types

**Pre-commit checklist**:

- [ ] `StatelessWidget` used (not `StatefulWidget`) unless UI-local state required
- [ ] `const` constructors in widget trees
- [ ] `ref.watch` used in build, `ref.read` in callbacks
- [ ] Widgets decomposed (no widget method >80 lines)
- [ ] Widget keys added for testability
- [ ] Riverpod providers use `@riverpod` annotation (code generation)
- [ ] shelf routes organized in handler classes
